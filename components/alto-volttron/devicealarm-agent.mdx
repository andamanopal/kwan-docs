## Overview

The DeviceAlarm Agent manages device-level alarms and alerts within the Alto Volttron platform. It monitors device status, processes alarm conditions, and manages notification workflows for equipment-specific issues.

## Configuration

### Basic Setup

```python
{
  "devices": {
    "chiller_01": {
      "enabled": true,
      "priority": "high",
      "alarm_points": [
        {
          "point": "status",
          "condition": "!=",
          "value": "running",
          "severity": "critical"
        },
        {
          "point": "temperature",
          "condition": ">",
          "value": 45,
          "severity": "warning"
        }
      ]
    }
  },
  "notifications": {
    "email": {
      "enabled": true,
      "recipients": {
        "critical": ["operations@example.com"],
        "warning": ["maintenance@example.com"]
      }
    },
    "sms": {
      "enabled": true,
      "critical_only": true,
      "recipients": ["+1234567890"]
    }
  }
}
```

## Usage Examples

### Configuring Alarms

```python
# Add new alarm configuration
agent.add_alarm(
    device_id="chiller_01",
    point_name="discharge_pressure",
    condition={
        "operator": ">",
        "value": 150,
        "severity": "critical",
        "delay": 30
    }
)
```

### Managing Notifications

```python
# Configure notification rules
agent.configure_notifications(
    device_id="chiller_01",
    rules={
        "critical": {
            "channels": ["email", "sms"],
            "escalation_delay": 300
        },
        "warning": {
            "channels": ["email"],
            "escalation_delay": 1800
        }
    }
)
```
